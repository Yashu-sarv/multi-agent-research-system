# 🔬 ResearchMind AI: Multi-Agent Research System

**ResearchMind AI** is a state-of-the-art research automation tool that leverages multiple specialized AI agents to extract, analyze, and synthesize information from the web into comprehensive professional reports. It features an intelligent **Smart Caching** system to optimize performance and reduce API costs.

---

## 🌟 Key Features
*   **Multi-Agent Orchestration:** Utilizes four specialized agents (Searcher, Reader, Writer, and Critic) working in a collaborative pipeline.
*   **Intelligent Database Caching:** Powered by **Supabase (PostgreSQL)**, the system caches every research report. Subsequent requests for the same topic are delivered instantly from the database.
*   **30-Day Expiration Policy:** Implements a cache freshness strategy where results older than 30 days trigger a fresh research cycle to ensure data accuracy.
*   **Deep Content Extraction:** Goes beyond search snippets by scraping full-text content from selected high-quality web resources.
*   **Automated Quality Assurance:** Includes an AI Critic that reviews and scores every report based on accuracy and depth.
*   **Modern Web Interface:** A sleek, responsive dashboard built with Streamlit for a premium user experience.

---

## 🤖 The Research Team (Agent Roles)
Built on **LangChain** and powered by **Mistral AI**, each agent has a specific mission:

1.  **🔍 The Searcher:** Scours the internet for the most relevant and up-to-date sources using the Tavily API.
2.  **📄 The Reader:** Performs deep scraping on selected URLs to extract raw data and detailed context.
3.  **✍️ The Writer:** Synthesizes collected information into a well-structured, professional Markdown report.
4.  **🧐 The Critic:** Evaluates the final draft for factual consistency, tone, and overall quality.

## 🚀 Getting Started

### 1. Prerequisites
Ensure you have the following API keys and credentials:
*   **Mistral AI API Key**
*   **Tavily Search API Key**
*   **Supabase Project URL & Anon Key**

### 2. Installation
```bash
git clone https://github.com/Yashu-sarv/multi-agent-research-system.git
cd multi-agent-research-system

# Create and activate virtual environment
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt
```

### 3. Database Configuration (Supabase)
Execute the following schema in your Supabase **SQL Editor**:
```sql
CREATE TABLE research_cache (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    topic TEXT UNIQUE,
    search_results TEXT,
    scraped_content TEXT,
    final_report TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Disable Row Level Security (RLS) for internal service access
ALTER TABLE research_cache DISABLE ROW LEVEL SECURITY;
```

### 4. Environment Variables
Create a `.env` file in the root directory:
```env
MISTRAL_API_KEY=your_mistral_key
TAVILY_API_KEY=your_tavily_key
SUPABASE_URL=your_project_url
SUPABASE_KEY=your_anon_key
```

### 5. Running the Application
```bash
streamlit run app.py
```
